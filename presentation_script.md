# Taxi Lucky Rate Dashboard Presentation Script

## 1. Intro & Input (Header & Route Form)
"안녕하세요, **Taxi Lucky Rate Dashboard** 발표를 시작하겠습니다.
이 대시보드는 뉴욕 택시 데이터를 분석하여, 특정 시간대에 택시를 타는 것이 얼마나 유리한지 알려주는 서비스입니다.

먼저 상단을 보시면 **출발지(Origin)**와 **도착지(Destination)**, 그리고 **시간(Time)**을 선택할 수 있는 입력 폼이 있습니다.
사용자가 원하는 경로와 시간을 입력하고 **'Analyze Route'** 버튼을 클릭하면, 서버에서 실시간으로 데이터를 분석하여 결과를 보여줍니다.
이때, 불필요한 서버 요청을 줄이기 위해 입력값이 바뀔 때마다 요청을 보내지 않고, 버튼을 클릭했을 때만 요청이 가도록 최적화했습니다."

## 2. TLI Score (Taxi Luck Index)
"분석 결과의 핵심인 **TLI(Taxi Luck Index)** 섹션입니다.
TLI는 해당 시간대의 교통 상황이 얼마나 '운이 좋은지'를 나타내는 지표입니다.
- **점수**: 0.0 이상의 값으로, 1.0을 초과할 수도 있습니다. 점수가 낮을수록(0.25 이하) 교통이 원활하고 예측 가능한 'Lucky'한 상태입니다.
- **시각화**: 원형 차트를 통해 직관적으로 점수를 확인할 수 있으며, 점수 구간에 따라 초록색(좋음), 주황색(보통), 빨간색(나쁨)으로 색상이 변합니다.
- **통계**: 옆에는 해당 경로의 **평균 소요 시간(Average Time)**과 **표준 편차(Std Deviation)**를 함께 보여주어, 예상 시간과 변동성을 한눈에 파악할 수 있습니다."

## 3. Congestion Index (CI Score)
"다음은 **CI(Congestion Index)**, 즉 혼잡도 지수입니다.
이 점수는 출발지와 도착지 자치구의 실시간 교통 혼잡도를 반영하여 계산됩니다.
- **로직**: 사용자가 선택한 시간대에 출발지와 도착지의 혼잡도 데이터를 각각 조회하고, 그 평균값을 계산하여 보여줍니다.
- **의미**: TLI가 '택시 잡기 좋은 정도'라면, CI는 '도로가 얼마나 막히는지'를 나타내는 보조 지표로 활용됩니다."

## 4. Timeline (Past & Future Trends)
"중앙의 **타임라인(Timeline)** 섹션입니다.
단순히 현재 시간만 보여주는 것이 아니라, **15분 간격**으로 앞뒤 데이터를 분석하여 TLI 변화 추이를 보여줍니다.
- **활용**: 만약 현재 시간의 TLI가 나쁘다면, 15분 뒤나 30분 뒤에는 상황이 나아지는지 비교해보고 출발 시간을 조정할 수 있습니다.
- **인터랙션**: 각 시간대 카드는 TLI 점수에 따라 색상이 구분되어 있어, 언제가 가장 좋은 시간대인지 직관적으로 알 수 있습니다."

## 5. Average Distance & Cost
"마지막으로 하단 영역입니다.
사용자가 선택한 경로의 **평균 이동 거리(Average Distance)**와 **예상 택시 요금(Average Cost)**을 보여줍니다.
이 데이터는 실제 과거 택시 운행 데이터를 기반으로 산출된 평균값으로, 사용자가 예산을 계획하는 데 도움을 줍니다."
